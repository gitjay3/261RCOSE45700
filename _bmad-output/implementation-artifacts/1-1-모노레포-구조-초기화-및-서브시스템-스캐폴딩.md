## Story

개발자로서,
서브시스템별 독립 의존성을 가진 모노레포 구조가 초기화되기를 원한다,
그래서 팀원이 각자 서브시스템(crawler/detection/api/dashboard)을 독립적으로 개발할 수 있다.

## Acceptance Criteria

1. **Given** 빈 git 저장소가 있을 때, **When** 모노레포 초기화 스크립트를 실행하면, **Then** `crawler/`, `detection/`, `api/`, `dashboard/`, `shared/`, `infra/`, `.github/workflows/` 디렉토리 구조가 프로젝트 루트에 생성된다. (Epic AC1 — `tracker/`는 프로젝트 루트 자체를 의미하며, 본 저장소의 루트가 `tracker` 역할을 수행한다)
2. `crawler/requirements.txt`에 다음 의존성이 정의된다: `playwright==1.58.0`, `APScheduler`, `beautifulsoup4`, `langdetect`, `redis`, `boto3`, `playwright-stealth`. (Epic AC2)
3. `detection/requirements.txt`에 다음 의존성이 정의된다: `redis`, `boto3`, `httpx`, `python-dotenv`. (Epic AC3)
4. `api/`는 **Spring Boot 3.4.x + Java 21** Gradle 프로젝트로 초기화되며, 의존성은 `web`, `data-jpa`, `postgresql`, `actuator`, `lombok`, `validation`이다. (Epic AC4)
5. `dashboard/`는 `npm create vite@latest -- --template react-ts`로 초기화되고, 다음 패키지가 설치된다: `@tanstack/react-query`, `axios`, `recharts`, `@radix-ui/react-select`, `date-fns`, `react-router-dom`. (Epic AC5)
6. `playwright install chromium`이 실행되어 Playwright Chromium 브라우저 바이너리가 설치되며, 실행 결과가 로그로 확인된다. (Epic AC6)

### Anti-Patterns to Avoid (이번 스토리 특화)

1. ❌ **중첩 `tracker/` 디렉토리 생성** — 저장소 루트가 곧 `tracker/`이다. `mkdir tracker/crawler` 절대 금지. `mkdir crawler` 사용.
2. ❌ **Spring Boot 4.0.5 사용** — architecture.md 일부 잔재 표기일 뿐, Epic AC와 Party Mode 결정은 3.4.x. `bootVersion=3.4.5` 사용.
3. ❌ **`shared/` 내부 파일을 미리 작성** — Story 1.2 범위. 디렉토리만 생성하고 `.gitkeep`만 둘 것.
4. ❌ **`infra/docker-compose.yml` 작성** — Story 1.3 범위.
5. ❌ **GitHub Actions 워크플로우 작성** — Story 1.5 범위. `.github/workflows/` 디렉토리만 생성.
6. ❌ **Spring `domain/`, `controller/` 등 빈 패키지 미리 생성** — Spring Initializr 기본 산출물만 둘 것. Epic 4에서 채움.
7. ❌ **dashboard에 `pages/`, `router.tsx` 등 미리 작성** — Vite 기본 템플릿(`App.tsx`, `main.tsx`)만 유지.
8. ❌ **글로벌 `.env`, `.env.local` 작성** — `.env.example`도 본 스토리에서는 placeholder만. 실제 환경변수 명세는 Story 1.3.
9. ❌ **`requirements.txt`에 미명시 라이브러리 추가** — Epic AC2/AC3에 명시된 정확한 목록만 포함. 추가 의존성 발견 시 별도 스토리.
10. ❌ **`playwright==1.58.0` 외 정확 핀 임의 추가** — pinning은 충분한 근거가 있을 때만.
