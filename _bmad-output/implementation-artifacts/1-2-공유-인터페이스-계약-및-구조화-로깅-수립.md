## Story

개발자로서,
크롤러·탐지·API 서브시스템이 공유하는 데이터 계약과 로깅 표준이 정의되기를 원한다,
그래서 팀원이 서로 독립적으로 개발하면서도 통합 시점에 스키마 충돌이 발생하지 않는다.

## Acceptance Criteria

1. **Given** `shared/pyproject.toml`이 `pip install -e shared/`를 지원하도록 설정될 때, **When** `pip install -e shared/`를 실행하면, **Then** `from shared.correlation_id import generate` 임포트가 성공한다. (Epic AC1)
2. **Given** `shared/` 모듈이 완성되면, **Then** `shared/correlation_id.py`가 UUID v4를 반환하는 `generate() -> str` 함수를 제공한다. (Epic AC2)
3. `shared/models/crawl_event.py`가 Redis MQ 메시지 스키마를 정의한다: `post_id`, `source_id`, `site_name`, `raw_text`, `image_urls`, `language`, `detected_at`, `correlation_id` 8개 필드. (Epic AC3)
4. `shared/interfaces/varco.py`가 `typing.Protocol` 클래스로 `translate(text: str) -> str`와 `classify(text: str) -> ClassificationResult`를 정의한다. `ClassificationResult`는 `is_illegal`, `type`, `confidence`, `reason` 4개 필드를 갖는다. (Epic AC4)
5. `shared/config/redis_config.py`가 DB 번호 상수를 정의한다: `REDIS_MQ_DB=0`, `REDIS_DEDUP_DB=1`, `REDIS_RATELIMIT_DB=2`, `REDIS_CACHE_DB=3`. (Epic AC5)
6. `shared/structured_logger.py`가 `{"timestamp", "service", "level", "correlation_id", "message"}` JSON 로그를 stdout에 출력하며, `service` 필드 값은 환경변수 `SERVICE_NAME`에서 읽는다. (Epic AC6)
7. `shared/exceptions/base_exception.py`가 공통 예외 기반 클래스를 정의한다. (Epic AC7)
8. `crawler/` 및 `detection/`의 `requirements.txt`에 `shared` 로컬 패키지 의존성이 추가되고(`-e shared/` 형태 또는 `pip install -e shared/` 방식), 각 서브시스템 venv에서 `from shared.models.crawl_event import CrawlEvent` 임포트가 성공한다.

### Project Context

- 저장소 루트 = `tracker/` 역할. `shared/`는 저장소 루트의 자식 디렉토리.
- **Story 1.1에서 `shared/` 하위 디렉토리 구조(`models/`, `interfaces/`, `config/`, `exceptions/`)는 `.gitkeep`만 있는 상태로 이미 생성됨.** 본 스토리에서 `.gitkeep`을 `__init__.py`와 실제 파일로 교체한다.
- `crawler/.venv`와 `detection/.venv` 가상환경은 Story 1.1에서 이미 생성되어 있음. 기존 venv를 재사용하고 `pip install -e shared/`만 추가 실행.
- Python 런타임: `3.11+` (architecture.md L93). `from __future__ import annotations`는 Python 3.11에서도 권장.

### Technical Stack Decisions

| 항목 | 결정 | 근거 |
|---|---|---|
| `CrawlEvent` 구현 방식 | `dataclasses.dataclass` | 표준 라이브러리, 추가 의존성 없음. `dataclasses.asdict()`로 Redis JSON 직렬화 지원. Pydantic은 외부 의존성이므로 shared/에 도입 시 모든 서브시스템이 pydantic에 묶임 — 오버엔지니어링 |
| `VarcoInterface` 구현 방식 | `typing.Protocol` | ABC(Abstract Base Class) 대비 덕타이핑 지원. `isinstance()` 체크를 위해 `@runtime_checkable` 추가. Story 1.4 varco_mock이 Protocol을 구현만 하면 되므로 상속 강제 없음 |
| `structured_logger` 기반 | Python 표준 `logging` | 외부 라이브러리(structlog, loguru) 배제. `logging.Formatter`를 재정의하여 JSON 출력. crawler/detection 서브시스템이 `pip install` 없이 바로 사용 |
| `pyproject.toml` 빌드 백엔드 | `setuptools>=68` | `pip install -e shared/` 지원. `flit`, `poetry`는 툴체인 확산 부작용. 표준 setuptools 충분 |
| `detected_at` 타입 | `str` (ISO 8601 UTC) | `datetime` 객체는 JSON 직렬화 시 명시적 변환 필요. architecture.md P5: `datetime.utcnow().isoformat() + "Z"` 표준으로 string 유지 |

### Architecture Compliance Notes

- **Correlation ID 전파** (architecture.md L62): `shared/correlation_id.py`는 Day 1 필수 선행 산출물. Crawler → Detection → API 3개 EC2가 동일 `correlation_id`를 Redis 메시지와 로그에 포함.
- **구조화 로깅 스키마** (architecture.md P6, L322~335): JSON 필드 5개 `{"timestamp", "service", "level", "correlation_id", "message"}` 엄수. `service` 허용값: `"crawler"`, `"detection"`, `"api"`. **하드코딩 금지, `SERVICE_NAME` 환경변수 주입 필수.**
- **Redis DB 분리** (architecture.md L64): `REDIS_MQ_DB=0, REDIS_DEDUP_DB=1, REDIS_RATELIMIT_DB=2, REDIS_CACHE_DB=3` — 단일 인스턴스 OOM 시 4기능 동시 장애 방지를 위한 논리 분리.
- **Python 필드 명명** (architecture.md P1): Redis MQ 메시지(`CrawlEvent`) 필드는 snake_case 유지 (`detected_at`, `post_id`). API JSON 경계에서만 camelCase 변환.
- **예외 처리** (architecture.md P10): `parse()` 류 함수에서 None 반환 금지. `TrackerBaseException` 상속 체인이 이 패턴을 강제함.

- `shared/structured_logger.py` 수동 검증:

### Previous Story Intelligence (Story 1.1)

- **Spring Boot 3.4.x → 3.5.0 변경 선례:** `requirements.txt`에 버전 고정 시 Spring Initializr처럼 외부 서비스가 버전을 거부하는 패턴 주의. `shared/pyproject.toml`의 `setuptools>=68` 도 최소 버전 핀 전략.
- **Foojay Toolchain Resolver / H2 선례:** Story 1.1에서 "스캐폴딩만" 했음에도 실제 실행(build) 시 추가 의존성 필요했음. `pip install -e shared/` 검증 시도 중 setuptools 관련 오류 발생 가능 → `pip install --upgrade pip setuptools` 선행 실행 권장.
- **`.gitkeep` → 실제 파일 교체 패턴:** Story 1.1이 `shared/`에 `.gitkeep`을 배치했음. 본 스토리는 `__init__.py`와 실제 구현 파일로 `.gitkeep` 제거 및 교체. `git rm shared/.gitkeep` 명령 필요.
- **Story 1.1 Review Deferred 항목 (`Python __init__.py만 있고 pyproject.toml 없음`):** 본 스토리가 직접 해소하는 항목. `from shared.models import ...`가 미해결 상태임을 확인하고 Task 1에서 해소.
- **저장소 루트가 `tracker/` 역할:** `pip install -e shared/` 실행 시 프로젝트 루트에서 실행. 상대 경로 `../shared` 대신 `shared/` (루트 기준 상대경로) 사용.

### Anti-Patterns to Avoid (이번 스토리 특화)

1. ❌ **Pydantic으로 `CrawlEvent` 정의** — 추가 의존성이 모든 서브시스템에 전파됨. `dataclasses.dataclass` 사용.
2. ❌ **`VarcoInterface`를 ABC로 구현** — `typing.Protocol`로 덕타이핑 지원이 올바른 패턴.
3. ❌ **`SERVICE_NAME` 하드코딩** — `structured_logger.py`에서 `"crawler"`, `"detection"` 등 서비스명을 소스코드에 직접 쓰지 말 것. `os.environ.get("SERVICE_NAME", "unknown")` 사용.
4. ❌ **`structured_logger`에 파일 핸들러 추가** — stdout 전용. Docker/EC2 환경에서 로그 수집기가 처리.
5. ❌ **`detection/src/mocks/varco_mock.py` 구현** — Story 1.4 범위. `VarcoInterface` Protocol만 정의하고 구현체는 만들지 말 것.
6. ❌ **`correlation_id`를 UUID 외 형식(int, short-uuid)으로 구현** — architecture.md L204: UUID 형식으로 3개 EC2 로그 추적.
7. ❌ **`CrawlEvent` 필드 추가/변경** — Epic AC3에 명시된 8개 필드만. 향후 필드 추가는 별도 스토리.
8. ❌ **`shared/` 내에 `requirements.txt` 작성** — `pyproject.toml`이 의존성 선언 담당. `shared` 자체는 외부 의존성 없는 표준 라이브러리만 사용.
