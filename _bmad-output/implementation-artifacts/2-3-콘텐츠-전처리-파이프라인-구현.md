# Story 2.3: crawl4ai 크롤러 개선 및 전처리 파이프라인 구현

Status: review

> 🎯 **본 스토리 핵심:** `crawler/`에 이미 있는 `Crawl4AICrawler`(`crawler/src/crawler.py`) + `SiteConfig` 레지스트리(`crawler/src/sites/registry.py`)를 기반으로, crawl4ai가 처리하지 않는 **언어 감지 · 중복 체크 · 키워드 필터** 3종 전처리 모듈을 추가하여, 통과한 게시글을 `shared/models/crawl_event.py` 스키마로 직렬화한다.  
>
> **[2026-04-28 피벗 반영]** Story 2.2의 `BaseSite`/`ProxyProvider` 추상화는 crawl4ai 전환으로 대체되어 **삭제됨**. 구 `crawler/` 코드(StealthBrowser, ProxyProvider, BaseSite, TailstarSite)는 제거되었고 `crawler2/` 코드베이스가 메인 `crawler/`로 승격됨. 신규 사이트 추가는 `SiteConfig` 레지스트리(`crawler/src/sites/registry.py`) 방식으로 진행. 프록시 교체 비용(NFR15)은 crawl4ai의 `BrowserConfig(proxy=...)` 단일 설정으로 대체.

## Story

개발자로서,
`crawler/`의 `Crawl4AICrawler` + `SiteConfig` 레지스트리를 기반으로 crawl4ai가 처리하지 않는 전처리(언어 감지·중복 체크·키워드 필터)가 추가되기를 원한다,
그래서 AI 탐지 큐에는 정제되고 언어가 식별된 관련 게시글만 전달된다.

## Acceptance Criteria

1. **Given** `crawler/src/crawler.py`의 `Crawl4AICrawler`가 있을 때 **When** `crawler/src/crawl4ai_crawler.py`로 리팩토링하면 **Then** `Crawl4AICrawler`를 제공하며, `AsyncWebCrawler(BrowserConfig(enable_stealth=True))` + `PruningContentFilter(threshold=0.5)` + httpx 이미지 다운로드가 동작한다 (FR2, FR3). `fetch()` 메서드는 `correlation_id` keyword-only 인자를 받으며, 크롤링 실패 시 `RuntimeError` 대신 `CrawlerException(CrawlerException.base_exception.CrawlerException)`을 raise한다.

2. **Given** `crawler/src/sites/registry.py`의 `SiteConfig` 레지스트리가 이미 있을 때 **When** `get_enabled_sites()` 함수 제공 여부를 확인하면 **Then** `SiteConfig` 데이터클래스와 `SITES` 딕셔너리, `get_enabled_sites()` 함수가 모두 존재한다. `inven_maple`이 기본 `enabled=True`로 등록된다 (FR1).

3. **Given** `crawler/requirements.txt`가 현재 `crawl4ai>=0.8.6`, `httpx>=0.27.0`을 포함할 때 **When** 전처리 의존성을 추가하면 **Then** `langdetect`가 `requirements.txt`에 추가된다. `crawl4ai-setup` 명령으로 patchright 브라우저를 설치해야 실제 크롤링이 동작하나, 단위 테스트는 mock으로만 검증한다.

4. **Given** `CrawlResult.fit_markdown` 텍스트가 있을 때 **When** `language_detector.detect(text, *, correlation_id)` 를 호출하면 **Then** `"ko"`, `"zh-CN"`, `"zh-TW"` 중 하나를 반환한다 (FR8). `langdetect` 라이브러리를 사용하며 `DetectorFactory.seed = 0`으로 결정론적 결과를 보장한다. 언어 감지 실패(`LangDetectException`) 시 `"ko"`를 기본값으로 반환하고 WARNING 로그를 기록한다. 결과는 `{"zh-cn": "zh-CN", "zh-tw": "zh-TW"}` 매핑으로 정규화한다.

5. **Given** Redis DB1(`REDIS_DEDUP_DB=1`)이 준비된 상태에서 **When** `dedup_checker.DedupChecker(redis_client).is_duplicate(text, *, correlation_id)` 를 호출하면 **Then** `hashlib.sha256(text.encode()).hexdigest()`를 계산하여 `posts:dedup` SET에 SISMEMBER로 조회한다 (FR9). 신규 게시글이면 `False` 반환. 이미 처리된 해시면 `True` 반환. `mark_seen(text, *, correlation_id)` 호출 시 SADD로 해시를 등록한다. 텍스트가 빈 문자열이면 `False`를 반환하고 dedup 등록하지 않는다.

6. **Given** 게시글의 `fit_markdown` 텍스트가 있을 때 **When** `keyword_filter.passes(text, *, correlation_id)` 를 호출하면 **Then** 불법 프로그램 관련 키워드(`매크로`, `핵`, `外挂`, `破解`, `辅助`, `텔레그램`, `오토`, `자동사냥`, `부스팅`, `hack`, `cheat`, `bot`, `exploit` 등 최소 10개)가 하나라도 포함된 경우 `True` 반환, 없으면 `False` 반환한다 (FR10). 대소문자 비구분 검색을 적용한다.

7. **Given** 전처리 3종(`language_detector` → `dedup_checker` → `keyword_filter`)을 통과한 게시글이 있을 때 **When** `CrawlEvent`를 생성하면 **Then** `shared/models/crawl_event.py` 스키마(`post_id`, `source_id`, `site_name`, `raw_text`=`result.markdown`, `image_urls`=`[img["src"] for img in result.images]`, `language`, `detected_at`=ISO 8601 UTC, `correlation_id`)가 완전히 채워진다. `post_id`는 `SiteConfig.post_id_extractor(url)` 결과를 사용하고, `source_id`는 `site_id` 문자열(레지스트리 키, 예: `"inven_maple"`)을 사용한다.

8. **Given** `Crawl4AICrawler`를 단위 테스트할 때 **When** `crawler/tests/unit/test_crawl4ai_crawler.py`를 실행하면 **Then** `AsyncWebCrawler`를 `unittest.mock.patch`로 mock하여 외부 네트워크 호출 0건으로 검증한다. 최소 검증 케이스: (a) 정상 크롤링 → `CrawlResult` 반환, (b) `result.success=False` → `CrawlerException` raise, (c) `fit_markdown` 없을 때 `raw_markdown` 폴백, (d) `image_filter` 콜백 적용, (e) score 임계값(≥3) 필터.

9. **Given** 단위 테스트를 실행할 때 **When** `cd crawler && ./.venv/bin/pytest -v`를 실행하면 **Then** 기존 25 테스트(Story 2.1~2.2) + 신규 테스트(`test_crawl4ai_crawler`, `test_keyword_filter`, `test_dedup_checker` 합계 ≥20건)가 **모두 PASS**하며 외부 네트워크 호출 0건이다.

> **AC 출처:** epics.md L334-L352 (Story 2.3). AC 1·7·8·9은 Story 2.2 패턴("correlation_id 전파", "StealthBrowser 비변경", "테스트 회귀 방지")을 crawl4ai 맥락으로 확장한 것이다.

### Anti-Patterns to Avoid

1. ❌ **`crawler/src/crawler.py` 직접 수정** — 원본 보존. 개선된 버전은 `crawl4ai_crawler.py`로 별도 작성.
2. ❌ **`crawler/src/storage.py`의 `PostStorage` 수정** — Story 2.4(S3 업로드) 범위. `crawl4ai_crawler.py`는 `downloaded_images: list[Path]`만 반환하고 저장 책임 없음.
3. ❌ **`dedup_checker.py`에서 Redis 직접 연결** — `redis.Redis(...)` 생성자 호출 금지. 생성자 DI 패턴 필수 (단위 테스트 mock 가능하도록).
4. ❌ **`language_detector.py`에서 실제 langdetect 호출로 테스트** — 비결정론적 동작 + 짧은 텍스트 오탐 방지를 위해 테스트에서는 mock 사용.
5. ❌ **`keyword_filter.py`에서 정규식 사용** — 단순 `in` 연산이 성능과 가독성 모두 우수. 정규식은 오탐/오검 위험 증가.
6. ❌ **APScheduler, `crawl_scheduler.py` 작성** — Story 2.5 범위. 본 스토리는 개별 모듈 단위까지만.
7. ❌ **Redis `LPUSH posts:queue` 호출** — Story 2.5 범위. 본 스토리는 `CrawlEvent` 객체 생성까지만.
8. ❌ **`test_crawl4ai_crawler.py`에서 실제 `AsyncWebCrawler` 호출** — crawl4ai-setup 브라우저 설치 없으면 CI 실패. 반드시 `patch("crawler.src.crawl4ai_crawler.AsyncWebCrawler")`.
9. ❌ **`serializer.py`에서 `datetime.utcnow()` 사용** — deprecated in Python 3.12+. `datetime.now(timezone.utc).isoformat()` 사용.
10. ❌ **`dedup_checker.py`에서 hash 함수로 `hash()` 사용** — PYTHONHASHSEED 환경변수에 따라 비결정론적. 반드시 `hashlib.sha256()`.

### Implementation Notes

1. **`__init__.py` 패키지 인식** — `crawler/src/preprocessor/__init__.py` 빈 파일 필수. 누락 시 `from crawler.src.preprocessor.language_detector import ...` 임포트 실패.
2. **pytest-asyncio 설정 필요** — `test_crawl4ai_crawler.py`의 모든 async 테스트에 `@pytest.mark.asyncio` 데코레이터 필수. `pytest.ini` 또는 `pyproject.toml`에 `asyncio_mode = auto` 또는 `strict` 설정 필요.
3. **`_SERVICE_NAME` 모듈 상수** — 신규 모듈 4개 모두에 `_SERVICE_NAME = os.environ.get("SERVICE_NAME", "crawler")` 추가.
4. **`correlation_id` keyword-only 패턴** — `fetch(url, *, correlation_id)` 형태로 통일.
5. **테스트 fixture 경로** — `tests/fixtures/html/sample_illegal_post.html` 접근 시 `Path(__file__).parent.parent.parent.parent / "tests" / "fixtures" / "html"` (crawler/tests/unit에서 4단계 상위).
6. **`hashlib.sha256` 사용** — `hash()` 내장 함수는 PYTHONHASHSEED에 따라 비결정적. dedup에서는 반드시 `hashlib.sha256()` 사용 (충돌 저항성).
7. **deferred-work.md의 `CrawlEvent.raw_text 크기 제한 없음`** — 이 스토리에서 수정하지 않는다. Story 2.5 부하 테스트 시 메시지 크기 상한 검토.

### Latest Tech Information (crawl4ai 0.8.6 기준)

- **crawl4ai 0.8.6** — patchright 기반 스텔스 브라우저 내장. `BrowserConfig(enable_stealth=True)` 한 줄로 Cloudflare JS 챌린지 우회 가능 (SPIKE 2.1 검증).
- **`PruningContentFilter(threshold=0.5, threshold_type="fixed")`** — 노이즈 블록(광고, 네비게이션) 제거 + LLM 최적화 Markdown 생성. `threshold_type="fixed"` 필수 (기본값 "dynamic"은 페이지마다 임계값 변동).
- **`result.markdown` vs `result.markdown.fit_markdown`** — crawl4ai 0.8.6에서 `result.markdown`은 `MarkdownGenerationResult` 객체. `.raw_markdown`, `.fit_markdown` 속성으로 접근. PoC의 `hasattr(md, "raw_markdown")` 분기 유지.
- **`CacheMode.BYPASS`** — 로컬 crawl4ai 캐시를 무시하고 항상 신규 요청. 프로덕션 크롤러에서 필수.
- **httpx 비동기 클라이언트** — `httpx.AsyncClient(timeout=15, follow_redirects=True)`. 이미지 다운로드 시 `Referer` 헤더 주입으로 403 방지 (PoC에서 검증).
- **langdetect 결정론적 모드** — `from langdetect import DetectorFactory; DetectorFactory.seed = 0`. 모듈 임포트 시점에 설정 필수 (테스트 환경 일관성).
