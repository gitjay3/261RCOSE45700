# Story 2.4: S3 원본 아카이브 및 이미지 업로드

Status: review

> 🎯 **본 스토리 핵심:** `Crawl4AICrawler`가 수집한 `fit_markdown` 텍스트와 `downloaded_images` 이미지 파일을 AWS S3에 아카이브한다. `S3Uploader` 신규 클래스로 업로드 로직을 캡슐화하고, `PostStorage`를 `ENABLE_S3_UPLOAD` 환경변수로 로컬 디스크 저장과 S3 업로드를 독립적으로 제어한다. `shared/models/crawl_event.py`에 S3 경로 필드(`s3_text_path`, `s3_image_paths`)를 선택적으로 추가하여 EC2 재시작 후 복구 가능한 원본 데이터 추적 체계를 완성한다.
>
> **[전제 조건]** Story 2.3이 `review` 상태. `crawl4ai_crawler.py`(`CrawlResult.downloaded_images: list[Path]`)와 `preprocessor/serializer.py`(`to_crawl_event()`)가 완성된 상태.

## Story

개발자로서,
크롤링된 텍스트(`fit_markdown`)와 이미지 파일이 S3에 보관되기를 원한다,
그래서 EC2 재시작 후에도 원본 데이터를 복구할 수 있다.

## Acceptance Criteria

1. **Given** 크롤링이 완료된 `CrawlResult`(`fit_markdown`, `downloaded_images: list[Path]`)가 있을 때 **When** `S3Uploader.upload_text(text, *, site, date, post_id, correlation_id)`를 호출하면 **Then** `fit_markdown` 텍스트가 `s3://{S3_BUCKET_NAME}/raw/{site}/{YYYY-MM-DD}/{post_id}.md` 경로로 업로드되고 해당 S3 URI 문자열을 반환한다 (FR5). `date` 파라미터는 `"YYYY-MM-DD"` 형식의 문자열이며, 호출 측에서 `datetime.now(timezone.utc).strftime("%Y-%m-%d")`로 생성한다.

2. **Given** `CrawlResult.downloaded_images`에 이미지 파일 목록(`list[Path]`)이 있을 때 **When** `S3Uploader.upload_images(image_paths, *, site, date, post_id, correlation_id)`를 호출하면 **Then** 각 파일이 `s3://{S3_BUCKET_NAME}/images/{site}/{YYYY-MM-DD}/{post_id}/{filename}` 경로로 업로드되고 S3 URI 목록(`list[str]`)을 반환한다 (FR4). `filename`은 `Path.name`으로 추출한다 (예: `img_000.jpg`).

3. **Given** `S3Uploader`가 초기화될 때 **When** boto3 클라이언트가 생성되면 **Then** `boto3.client("s3", region_name=AWS_REGION)` 호출 시 Access Key 또는 Secret Key를 코드에 직접 전달하지 않으며, EC2 IAM Role의 instance metadata를 통해 자동으로 자격증명이 제공된다 (NFR6). `AWS_REGION` 환경변수가 없을 때는 boto3 기본값을 사용한다.

4. **Given** S3 업로드 중 에러가 발생할 때 **When** `botocore.exceptions.ClientError`가 raise되면 **Then** 이를 그대로 전파하며 구조화 로그에 `correlation_id`와 함께 ERROR 레벨로 기록한다. 개별 이미지 업로드 실패 시 해당 이미지 로그 후 다음 이미지로 진행하고 실패한 이미지는 반환 목록에서 제외한다. 텍스트 업로드 실패 시는 즉시 예외를 raise한다.

5. **Given** 전처리를 통과한 게시글의 `CrawlEvent`가 생성될 때 **When** `shared/models/crawl_event.py`의 `CrawlEvent`가 사용되면 **Then** `s3_text_path: str = ""`와 `s3_image_paths: list[str] = []` 필드가 선택적으로 포함된다. S3 업로드가 활성화(`ENABLE_S3_UPLOAD=true`)된 경우 해당 필드에 실제 S3 URI가 채워진다. `CrawlEvent.from_json()`은 기존 직렬화 데이터(s3 필드 없음)를 하위 호환 처리한다. `_CRAWL_EVENT_FIELDS` frozenset에 두 필드가 추가된다.

6. **Given** 환경변수 `ENABLE_S3_UPLOAD`가 설정된 환경에서 **When** `PostStorage.save(*, site_id, post_id, url, result, correlation_id)`가 실행되면 **Then** `ENABLE_S3_UPLOAD=true`일 때 로컬 디스크 저장과 S3 업로드가 모두 실행되고 `StorageResult(local_path, s3_text_path, s3_image_paths)`를 반환한다. `ENABLE_S3_UPLOAD=false`(기본값) 또는 미설정 시 로컬 저장만 실행하고 `StorageResult(local_path, s3_text_path="", s3_image_paths=[])`를 반환한다. `PostStorage` 생성자에서 `ENABLE_S3_UPLOAD` 및 `S3_BUCKET_NAME` 환경변수를 읽고, `ENABLE_S3_UPLOAD=true`이면 내부적으로 `S3Uploader`를 생성한다.

7. **Given** 단위 테스트를 실행할 때 **When** `cd crawler && ./.venv/bin/pytest -v`를 실행하면 **Then** 기존 27 테스트(Story 2.1~2.3) + 신규 `test_s3_uploader.py` ≥10건이 **모두 PASS**하며 실제 boto3 S3 호출 0건이다. boto3는 `unittest.mock.patch`로 mock한다.

### boto3 IAM Role 패턴 (NFR6)

```python
import os
import boto3

class S3Uploader:
    def __init__(self, bucket_name: str) -> None:
        self._bucket = bucket_name
        # ✅ 올바른 패턴 — IAM Role 자동 사용
        region = os.environ.get("AWS_REGION")
        kwargs = {"region_name": region} if region else {}
        self._client = boto3.client("s3", **kwargs)

    def upload_text(self, text: str, *, site: str, date: str, post_id: str, correlation_id: str) -> str:
        key = f"raw/{site}/{date}/{post_id}.md"
        try:
            self._client.put_object(
                Bucket=self._bucket,
                Key=key,
                Body=text.encode("utf-8"),
                ContentType="text/markdown; charset=utf-8",
            )
            _logger.info(
                "S3 텍스트 업로드 완료: s3://%s/%s", self._bucket, key,
                extra={"correlation_id": correlation_id, "service": _SERVICE_NAME},
            )
        except botocore.exceptions.ClientError as exc:
            _logger.error(
                "S3 텍스트 업로드 실패: %s — %s", key, exc,
                extra={"correlation_id": correlation_id, "service": _SERVICE_NAME},
            )
            raise
        return f"s3://{self._bucket}/{key}"
```

### CrawlEvent 수정 패턴

```python
# shared/models/crawl_event.py 수정 핵심

_CRAWL_EVENT_FIELDS = frozenset({
    "post_id", "source_id", "site_name", "raw_text",
    "language", "detected_at", "correlation_id", "image_urls",
    "s3_text_path", "s3_image_paths",   # Story 2.4 추가 (선택 필드)
})

@dataclass
class CrawlEvent:
    post_id: str
    source_id: str
    site_name: str
    raw_text: str
    language: str
    detected_at: str
    correlation_id: str
    image_urls: list[str] = field(default_factory=list)
    s3_text_path: str = ""                              # Story 2.4 추가
    s3_image_paths: list[str] = field(default_factory=list)  # Story 2.4 추가

    @classmethod
    def from_json(cls, data: str) -> CrawlEvent:
        payload = json.loads(data)
        unknown = set(payload) - _CRAWL_EVENT_FIELDS
        if unknown:
            raise ValueError(f"CrawlEvent: unknown fields in payload: {unknown}")
        # 하위 호환: image_urls, s3_text_path, s3_image_paths는 선택 필드
        missing = _CRAWL_EVENT_FIELDS - {"image_urls", "s3_text_path", "s3_image_paths"} - set(payload)
        if missing:
            raise ValueError(f"CrawlEvent: missing required fields: {missing}")
        if not isinstance(payload.get("image_urls", []), list):
            raise ValueError("CrawlEvent: 'image_urls' must be a list")
        return cls(**{k: v for k, v in payload.items() if k in _CRAWL_EVENT_FIELDS})
```

### Anti-Patterns to Avoid

1. ❌ **boto3 자격증명 하드코딩** — `aws_access_key_id`, `aws_secret_access_key` 파라미터 전달 절대 금지. EC2 IAM Role 사용 (NFR6).
2. ❌ **`upload_text()` 실패 시 silent failure** — `ClientError`는 반드시 재raise. 텍스트 업로드 실패는 치명적이므로 호출측에서 처리하게 한다.
3. ❌ **`storage.py` 수정 없이 S3Uploader 단독 사용** — PostStorage가 S3Uploader를 감싸서 `ENABLE_S3_UPLOAD` 환경변수로 제어해야 함. 직접 S3Uploader를 호출하는 코드를 crawler 메인 로직에 넣지 않는다.
4. ❌ **CrawlEvent에서 s3 필드를 필수(required)로 만들기** — `ENABLE_S3_UPLOAD=false` 환경에서도 `CrawlEvent`를 사용하므로 반드시 선택 필드(`default=""`, `default_factory=list`)로 추가.
5. ❌ **실제 boto3 호출로 단위 테스트** — `unittest.mock.patch`로 `boto3.client` mock 필수. 실제 AWS 연결은 CI 실패 원인.
6. ❌ **`datetime.utcnow()` 사용** — Python 3.12+에서 deprecated. `datetime.now(timezone.utc).strftime("%Y-%m-%d")` 사용.
7. ❌ **`PostStorage` 생성자에서 `ENABLE_S3_UPLOAD` 하드코딩** — 반드시 `os.environ.get("ENABLE_S3_UPLOAD", "false").lower() == "true"` 패턴으로 환경변수 읽기.
8. ❌ **이미지 업로드 전체 실패 시 파이프라인 중단** — 이미지는 텍스트보다 중요도가 낮고 CDN 다운 등 일시적 원인일 수 있음. 개별 실패는 WARNING 로그 후 continue.
9. ❌ **`s3_text_path`를 `None`으로 초기화** — 빈 문자열 `""` 사용. `None` 사용 시 Story 2.5의 Redis 직렬화에서 타입 오류 발생.

### Implementation Notes

1. **`StorageResult` 데이터클래스 위치** — `crawler/src/storage.py` 파일 상단에 정의. `S3Uploader`와 동일 파일에 두지 않는다 (순환 임포트 방지).
2. **`PostStorage` 내부 S3Uploader 생성 vs DI** — Story 2.4에서는 `PostStorage.__init__()`에서 환경변수를 읽어 내부적으로 `S3Uploader`를 생성한다. 단위 테스트에서 `patch("crawler.src.storage.S3Uploader")` 사용. 직접 DI가 필요하면 Story 2.5에서 리팩토링.
3. **`boto3` 임포트 위치** — `s3_uploader.py` 파일에서만 임포트. `storage.py`는 타입 힌트용 `TYPE_CHECKING` 블록으로 S3Uploader를 참조해도 됨.
4. **`S3_BUCKET_NAME` 누락 처리** — `ENABLE_S3_UPLOAD=true`이지만 `S3_BUCKET_NAME`이 없으면 `PostStorage.__init__()`에서 `ValueError: S3_BUCKET_NAME is required when ENABLE_S3_UPLOAD=true`를 raise. 런타임에 조용히 실패하지 않도록.
5. **로컬 저장 완료 후 S3 업로드** — 로컬 `post.json` 저장이 완료된 후 S3 업로드 시도. 로컬 저장 실패는 기존대로 예외 전파.
6. **이미지 파일 경로 확인** — `upload_images()` 호출 시 `image_paths`에 실제 존재하는 파일 경로만 전달. `PostStorage.save()`에서 로컬 저장 완료 후 생성된 `post_dir / r["filename"]` 경로를 사용하므로 파일 존재 보장.
7. **`test_s3_uploader_no_hardcoded_credentials` 구현** — `boto3.client` 호출의 `kwargs`에 `aws_access_key_id`, `aws_secret_access_key`가 없음을 `call_args` 검사로 검증:
   ```python
   call_kwargs = mock_boto3.client.call_args.kwargs
   assert "aws_access_key_id" not in call_kwargs
   assert "aws_secret_access_key" not in call_kwargs
   ```
