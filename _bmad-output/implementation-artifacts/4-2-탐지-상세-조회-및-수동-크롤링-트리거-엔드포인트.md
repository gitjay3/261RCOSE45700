# Story 4.2: 탐지 상세 조회 및 수동 크롤링 트리거 엔드포인트

Status: done

## Story

백엔드 개발자로서,
탐지 상세 조회와 수동 크롤링 트리거 API가 구현되기를 원한다,
그래서 담당자가 특정 게시글 상세 정보를 확인하고 긴급 시 즉시 크롤링을 실행할 수 있다.

## Acceptance Criteria

1. **Given** 유효한 `detection_id`가 있을 때 **When** `GET /api/detections/{id}`를 요청하면 **Then** 원문(`rawText`), 번역문(`translatedText`·중국어인 경우), 탐지 유형(`type`), 신뢰도(`confidence`), 판단 근거(`reason`), 출처 URL(`postUrl`)을 포함한 `DetectionResponse`가 반환된다 (FR20, FR21)
2. **And** 존재하지 않는 `id`에 대해 ProblemDetail 형식의 `404 DETECTION_NOT_FOUND`를 반환한다 (architecture.md P4)
3. **When** `POST /api/crawl/trigger`를 요청하면 **Then** Redis DB0 `crawl:trigger` 채널에 메시지를 PUBLISH하고 `202 Accepted`와 함께 `{"status": "triggered", "estimatedMinutes": 3}`을 반환한다 (FR6)
4. **And** `GET /api/detections/{id}`와 `POST /api/crawl/trigger` 응답 헤더에 `X-Correlation-ID`가 포함된다
5. **And** `api/src/test/java/com/tracker/api/controller/DetectionControllerTest.java`에서 상세 조회(200)·존재하지 않는 ID(404)·크롤 트리거(202) 케이스를 포함해 `./gradlew test`가 통과한다

### Task 10 상세: CrawlTriggerService

```java
package com.tracker.api.service;

import lombok.RequiredArgsConstructor;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class CrawlTriggerService {

    private static final String CRAWL_TRIGGER_CHANNEL = "crawl:trigger";

    private final StringRedisTemplate stringRedisTemplate;

    public void trigger(String correlationId) {
        stringRedisTemplate.convertAndSend(CRAWL_TRIGGER_CHANNEL, correlationId);
    }
}
```

**중요:** `StringRedisTemplate` — Spring Boot가 `spring-boot-starter-data-redis` 의존성과 `spring.data.redis.*` 설정으로 자동 구성. `RedisTemplate<String, String>` 대신 `StringRedisTemplate` 사용 (키·값 모두 String 직렬화, pub/sub 메시지 타입 일치). 채널 값 `"crawl:trigger"`은 `shared/config/redis_config.py`의 `REDIS_CHANNEL_CRAWL_TRIGGER`와 동일해야 함.

Redis PUBLISH 실패 시 예외가 `CrawlController`로 전파 → `GlobalExceptionHandler`의 `handleAll()` fallback → 500 `INTERNAL_SERVER_ERROR` 반환.

### Task 11 상세: CrawlController

```java
package com.tracker.api.controller;

import com.tracker.api.dto.CrawlTriggerResponse;
import com.tracker.api.service.CrawlTriggerService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import jakarta.servlet.http.HttpServletRequest;
import lombok.RequiredArgsConstructor;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import java.util.UUID;

@RestController
@RequestMapping("/api")
@RequiredArgsConstructor
@Tag(name = "Crawl", description = "수동 크롤링 트리거")
public class CrawlController {

    private final CrawlTriggerService crawlTriggerService;

    @PostMapping("/crawl/trigger")
    @Operation(summary = "수동 크롤링 트리거", description = "Redis crawl:trigger 채널에 PUBLISH. 완료까지 약 3분 소요.")
    public ResponseEntity<CrawlTriggerResponse> trigger(HttpServletRequest request) {

        String correlationId = request.getHeader("X-Correlation-ID");
        if (correlationId == null || correlationId.isBlank()) {
            correlationId = UUID.randomUUID().toString();
        }

        crawlTriggerService.trigger(correlationId);

        return ResponseEntity.accepted()
                .header("X-Correlation-ID", correlationId)
                .body(new CrawlTriggerResponse("triggered", 3));
    }
}
```

`@RequestMapping("/api")` 유지 — 전체 경로 `/api/crawl/trigger`. `ResponseEntity.accepted()` = 202 Accepted.

### Anti-Patterns to Avoid

1. ❌ **`findById()` 직접 사용** — Spring Data의 기본 `findById()`는 `post`, `source`를 LAZY로 가져오므로 `DetectionResponse.from(d)` 호출 시 `LazyInitializationException` 발생. 반드시 `findByIdFetched()` 커스텀 쿼리 사용.

2. ❌ **`@Transactional` 없이 `findByIdFetched()` 호출** — Service 메서드에 `@Transactional(readOnly = true)` 필수. 트랜잭션 없으면 LAZY 탐색 불가.

3. ❌ **`RedisTemplate<Object, Object>` 사용** — `StringRedisTemplate`을 사용해야 한다. `convertAndSend()`는 String 직렬화. `RedisTemplate`의 기본 직렬화는 `JdkSerializationRedisSerializer`로 바이트 배열 전송 → Crawler `trigger_listener.py`에서 파싱 오류 발생.

4. ❌ **DB0 이외의 Redis DB 사용** — `crawl:trigger` 채널은 반드시 DB0 (`REDIS_MQ_DB=0`). Crawler의 `trigger_listener.py`가 `REDIS_MQ_DB=0`에서 구독 중. `application.properties`에 `spring.data.redis.database=0` 명시 필수.

5. ❌ **`ResponseEntity.ok()` 반환** — `POST /api/crawl/trigger`는 `202 Accepted` 반환. `ResponseEntity.accepted()` 사용.

6. ❌ **`@WebMvcTest(DetectionController.class)` 유지** — `CrawlController` 테스트 추가 시 `@WebMvcTest({DetectionController.class, CrawlController.class})`로 변경 필수. `@MockitoBean CrawlTriggerService` 누락 시 `UnsatisfiedDependencyException` 발생.

7. ❌ **`@MockBean` 사용** — Spring Boot 3.4+에서 deprecated. 기존 코드와 동일하게 `@MockitoBean` 사용.

8. ❌ **`application-test.properties`에 Redis 설정 누락** — `spring-boot-starter-data-redis` 추가 후 `@SpringBootTest` 실행 시 Redis 연결 시도. `application-test.properties`에 autoconfigure exclude 추가 필수. 누락 시 `TrackerApiApplicationTests` 실패.

9. ❌ **`DetectionNotFoundException` 응답에 스택 트레이스 포함** — `GlobalExceptionHandler`의 패턴 동일 적용. `ex.getMessage()`만 `detail`로 전달, 스택 트레이스 응답 금지.

10. ❌ **`@RequestMapping("/api/crawl")` 사용** — 반드시 `@RequestMapping("/api")` + `@PostMapping("/crawl/trigger")` 조합. 프론트엔드 `client.ts`의 baseURL `/api` 기준과 Swagger 문서화를 위해 `/api` prefix 유지.
