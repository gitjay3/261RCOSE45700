# Story 4.3: 통계 엔드포인트 구현

Status: done

## Story

백엔드 개발자로서,
오늘 탐지 수·유형별·사이트별·언어별 분포와 주간·월간 추이를 반환하는 `GET /stats` API가 구현되기를 원한다,
그래서 팀장이 보고 자료용 통계 데이터를 즉시 조회할 수 있다.

## Acceptance Criteria

1. **Given** RDS `detections`에 복수의 탐지 결과가 있을 때 **When** `GET /api/stats`를 요청하면 **Then** 오늘 총 탐지 수와 전일 대비 증감(`todayCount`, `deltaFromYesterday`)이 반환된다 (FR23)
2. **And** 탐지 유형별 분포(`typeDistribution: [{type, count}]`)가 반환된다 (FR24)
3. **And** 사이트별 분포(`siteDistribution: [{site, count}]`)가 반환된다 (FR25)
4. **And** 언어별 분포(`langDistribution: [{lang, count}]`)가 반환된다 (FR27)
5. **And** `?period=weekly` 또는 `?period=monthly` 파라미터로 추이 데이터(`trend: [{date, count}]`)가 반환된다 (FR26)
6. **And** Redis DB3(`cache:detections`)에 통계 결과가 캐싱되어 반복 요청 시 RDS 쿼리 없이 응답한다
7. **And** `api/src/test/java/com/tracker/api/controller/StatsControllerTest.java`에서 정상 응답·캐시 히트·period 파라미터 검증 케이스를 포함해 `./gradlew test`가 통과한다

### RedisConfig.java 구현 가이드

Story 4.2에서 `spring.data.redis.database=0`으로 DB0이 기본 연결로 설정되어 있다. DB3 캐시용 Bean은 **별도 `LettuceConnectionFactory`**를 생성해야 한다. `StringRedisTemplate`(또는 `RedisTemplate<String,String>`)의 기본 오토컨피그 Bean을 오버라이드하면 `CrawlTriggerService`의 DB0 pub/sub이 깨진다. 반드시 `@Qualifier("cacheRedisTemplate")`로 분리.

```java
package com.tracker.api.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.lettuce.LettuceConnectionFactory;
import org.springframework.data.redis.core.StringRedisTemplate;

@Configuration
public class RedisConfig {

    @Value("${spring.data.redis.host:localhost}")
    private String redisHost;

    @Value("${spring.data.redis.port:6379}")
    private int redisPort;

    @Value("${spring.data.redis.cache.database:3}")
    private int cacheDb;

    @Bean("cacheRedisTemplate")
    public StringRedisTemplate cacheRedisTemplate() {
        var cf = new LettuceConnectionFactory(redisHost, redisPort);
        cf.setDatabase(cacheDb);
        cf.afterPropertiesSet();
        return new StringRedisTemplate(cf);
    }
}
```

`application.properties`에 추가:
```properties
# Redis DB3 — API 통계 캐시 (REDIS_CACHE_DB=3)
spring.data.redis.cache.database=${REDIS_CACHE_DB:3}
```

**중요:** `application-test.properties`의 `spring.autoconfigure.exclude`는 이미 Redis AutoConfiguration을 제외하고 있으므로, `@WebMvcTest`·`@SpringBootTest` 모두 `@MockitoBean(name="cacheRedisTemplate") StringRedisTemplate` 또는 `@MockitoBean StatsService`로 커버 가능.

### StatsRepository JPQL 가이드

PostgreSQL에서 날짜 추출: JPQL에서 `FUNCTION('DATE_TRUNC', 'day', d.detectedAt)` 또는 네이티브 쿼리 사용. H2 테스트 환경이 아닌 `@SpringBootTest` 없이 `@WebMvcTest` + `@MockitoBean StatsService` 구조로 테스트하므로 **StatsRepository는 테스트 환경에서 실행되지 않는다**. 따라서 H2 호환성 고려 불필요 — PostgreSQL JPQL만 정확히 작성하면 된다.

Projection 사용 시: `Object[]` 배열 또는 별도 `@SqlResultSetMapping` 없이 `List<Object[]>` 반환 후 `StatsService`에서 매핑 권장.

```java
// 유형별 분포 예시
@Query("""
    SELECT d.type AS label, COUNT(d) AS cnt
    FROM Detection d
    WHERE d.confidence >= 0.70
    GROUP BY d.type
    ORDER BY COUNT(d) DESC
    """)
List<Object[]> findTypeDistributionRaw();
```

`StatsService`에서:
```java
var typeDistribution = statsRepository.findTypeDistributionRaw().stream()
    .map(row -> new StatsResponse.TypeDistributionItem((String) row[0], (Long) row[1]))
    .toList();
```

### 캐시 직렬화 전략

`StringRedisTemplate`이므로 Redis value는 JSON 문자열. `ObjectMapper`(Spring Boot 오토컨피그 Bean)를 `StatsService`에 주입하여 `StatsResponse` ↔ JSON 문자열 변환.

```java
// 캐시 저장
String json = objectMapper.writeValueAsString(statsResponse);
cacheRedisTemplate.opsForValue().set(cacheKey, json, Duration.ofSeconds(60));

// 캐시 조회
String cached = cacheRedisTemplate.opsForValue().get(cacheKey);
if (cached != null) {
    return objectMapper.readValue(cached, StatsResponse.class);
}
```

캐시 키 네이밍: `"cache:detections:stats"` (period 없음), `"cache:detections:stats:weekly"`, `"cache:detections:stats:monthly"`.

### StatsController 구현 가이드

`DetectionController`·`CrawlController` 패턴과 완전히 동일:

```java
package com.tracker.api.controller;

import com.tracker.api.dto.StatsResponse;
import com.tracker.api.service.StatsService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import jakarta.servlet.http.HttpServletRequest;
import lombok.RequiredArgsConstructor;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import java.util.UUID;

@RestController
@RequestMapping("/api")
@RequiredArgsConstructor
@Tag(name = "Stats", description = "탐지 통계 조회")
public class StatsController {

    private final StatsService statsService;

    @GetMapping("/stats")
    @Operation(summary = "탐지 통계 조회",
               description = "오늘 탐지 현황 및 유형/사이트/언어별 분포. period=weekly|monthly 로 추이 포함.")
    public ResponseEntity<StatsResponse> getStats(
            @RequestParam(required = false) String period,
            HttpServletRequest request) {

        StatsResponse stats = statsService.getStats(period);

        String correlationId = request.getHeader("X-Correlation-ID");
        if (correlationId == null || correlationId.isBlank()) {
            correlationId = UUID.randomUUID().toString();
        }

        return ResponseEntity.ok()
                .header("X-Correlation-ID", correlationId)
                .body(stats);
    }
}
```

### Anti-Patterns to Avoid

1. ❌ **기본 `StringRedisTemplate` Bean 오버라이드** — `@Bean StringRedisTemplate`을 `@Primary`로 등록하면 `CrawlTriggerService` DB0 pub/sub 채널이 DB3으로 대체됨. 반드시 `@Bean("cacheRedisTemplate")`으로 이름 지정 후 `@Qualifier("cacheRedisTemplate")`로 주입.

2. ❌ **`spring.data.redis.database` 값 변경** — 이 값은 DB0 전용(`CrawlTriggerService`). DB3 캐시를 위해 이 값을 3으로 변경하면 `POST /api/crawl/trigger`가 broken됨. 별도 프로퍼티 `spring.data.redis.cache.database=3` 사용.

3. ❌ **H2에서 실행되는 StatsRepository 쿼리 작성** — 테스트 전략이 `@WebMvcTest` + `@MockitoBean StatsService`이므로, StatsRepository는 테스트 환경에서 실행되지 않음. H2 호환 쿼리를 위해 PostgreSQL 네이티브 쿼리를 제한할 필요 없음.

4. ❌ **캐시 TTL 무한** — 반드시 60초 TTL. `cacheRedisTemplate.opsForValue().set(key, json)`만 호출하면 TTL 없이 영구 저장됨. `set(key, json, Duration.ofSeconds(60))` 3-arg 버전 사용.

5. ❌ **Redis 장애 시 예외 전파** — 캐시 레이어 실패는 비즈니스 로직과 무관. try-catch로 Redis 예외를 잡아 로깅 후 RDS fallback 필수. 캐시 실패가 `500 INTERNAL_SERVER_ERROR` 응답을 유발하면 안 됨.

6. ❌ **`@MockBean` 사용** — Spring Boot 3.4+에서 deprecated. `@MockitoBean` 사용 (기존 테스트 파일과 동일).

7. ❌ **`confidence < 0.70` 데이터 포함** — 모든 통계 쿼리에 `d.confidence >= 0.70` 필터 필수. 프론트엔드는 신뢰도 0.70 미만 데이터를 표시하지 않는 것으로 구현되어 있음 (FR22 일관성).

8. ❌ **`trend` 날짜 포맷 불일치** — `TrendItem.date`는 `"2026-04-24"` 형식 ISO 날짜 문자열. `Instant.toString()`이나 DateTime 전체 반환 금지. `String.format("%tF", date)` 또는 `date.toString()` (LocalDate → "yyyy-MM-dd").

9. ❌ **`@RequestMapping("/api/stats")` 사용** — 반드시 `@RequestMapping("/api")` + `@GetMapping("/stats")`. 기존 컨트롤러 일관성 유지.
