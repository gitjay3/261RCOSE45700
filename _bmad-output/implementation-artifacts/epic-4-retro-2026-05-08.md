## 2. 잘 된 것들

### A. 프론트-백엔드 API 계약 정합성

- MSW mock(`handlers.ts`)을 프론트에서 먼저 구현하고, 백엔드가 동일한 구조로 맞춰 구현
- `DetectionResponse`, `StatsResponse` 필드가 프론트 `types/api.ts`와 1:1 일치
- 4-3 리뷰에서 필드명 불일치 1건만 발생 — 즉시 패치 처리
- **결과: 프론트-백엔드 통합 마찰 최소화**

### B. Scope Boundary 테이블로 스토리 간 경계 명확화

- 각 백엔드 스토리에 '이번 스토리에서 한다 / 하지 않는다' 테이블 명시
- 4-1 → Redis 제외, 4-2 → Stats 제외, 4-3 → 프론트 연동 제외 형태로 역할 분리 명확
- 범위 충돌 없이 3개 스토리 순차 완료

### C. 테스트 전략의 점진적 성숙

- 4-1: `@WebMvcTest` + `@MockitoBean` 패턴 정착
- 4-2: Redis 채널 계약 테스트(`CrawlTriggerServiceTest`) 추가
- 4-3: `StatsServiceTest` — 캐시 hit/miss/fallback 3종 커버
- Tracker 직접 평가: **"테스트 케이스를 잘 작성한 게 좋았다"**

### D. 발견한 기술 문제의 즉각 문서화 및 재사용

- PostgreSQL JPQL 파라미터 타입 추론 버그(Hibernate 6) → `cast()` 구문으로 해결, 4-2/4-3에서 동일 패턴 재사용
- Redis DB 분리 패턴(DB0 pub/sub / DB3 cache) → `@Qualifier` + 명시적 생성자 패턴 확립
- `X-Correlation-ID` 에코 패턴 → 3개 컨트롤러 전체 재사용

## 3. 어려웠던 것들

### E. 인프라 의존성 추가 시 테스트 환경 설정 동기화 문제

- Story 4-2에서 `spring-boot-starter-data-redis` 추가 후 `@SpringBootTest`(`TrackerApiApplicationTests`) 깨짐
- Redis 연결 시도 → `application-test.properties`에 `spring.autoconfigure.exclude` 추가로 해결
- `@WebMvcTest`는 Redis를 자동 제외하지만 `@SpringBootTest`는 그렇지 않다는 비직관적 차이
- **Tracker 직접 체감: "테스트 환경 제외 문제가 체감이 컸다"**
- 이 설정이 4-3 dev notes에 '변경 금지'로 명시돼야 했음

### F. Redis 멀티 DB 분리의 예상 외 복잡성

- DB0(pub/sub)과 DB3(cache)를 분리하려면 별도 `LettuceConnectionFactory` + `@Qualifier("cacheRedisTemplate")` 필수
- `@RequiredArgsConstructor`와 `@Qualifier` 불호환 → 명시적 생성자 작성 필요
- deprecated `LettuceConnectionFactory.setDatabase()` 대신 `RedisStandaloneConfiguration.setDatabase()` 사용해야 함

### G. architecture.md 버전 표기 오류 반복

- architecture.md: Spring Boot `3.4.x` 기재 ↔ 실제: `3.5.0`
- 3개 스토리(4-1, 4-2, 4-3) dev notes 전체에 '주의사항'으로 반복 등장
- 한 번 수정했으면 없었을 반복 비용

### H. 이전 retro 액션 아이템 미처리

- AI-1(Today timestamp 자정 롤오버), AI-2(3-column 레이아웃 desktop-only 명시) — '즉시 처리 가능'으로 분류됐으나 미처리로 이월
- 이전 회고 약속 이행률: 6건 중 1건 완료(AI-6 부분 해소)

## Appendix: 백엔드 주요 기술 결정

| 결정 | 내용 | 스토리 |
|---|---|---|
| JPQL 파라미터 타입 명시 | `cast(:param as Type)` — PostgreSQL Hibernate 6 버그 대응 | 4-1 |
| `boolean illegal` 필드명 | Lombok `isIsIllegal()` 버그 회피 | 4-1 |
| `@MockitoBean` (not `@MockBean`) | Spring Boot 3.4+ deprecated 대응 | 4-1~4-3 전체 |
| Redis AutoConfig 제외 | `application-test.properties` exclude 패턴 | 4-2 |
| `StringRedisTemplate` (not `RedisTemplate`) | crawl:trigger 채널 String 직렬화 계약 | 4-2 |
| `@Bean("cacheRedisTemplate")` 분리 | DB0/DB3 분리, `@Qualifier` 명시적 생성자 | 4-3 |
| `RedisStandaloneConfiguration.setDatabase()` | deprecated `LettuceConnectionFactory.setDatabase()` 대신 | 4-3 |
| `period` 파라미터 검증 + 0건 날짜 채움 | invalid period → 400, trend 7/30일 빈 날짜 zero-fill | 4-3 |
